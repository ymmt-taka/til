# rails gコマンド Dockerを使用してRailsアプリケーションを開発する場合、Railsのgeneratorを介してファイルを生成するコマンド。 `rails g model`：新しいモデルとマイグレーションファイルを生成 `rails g migration`：データベースの変更を行う新しいマイグレーションを生成する。 `rails g controler`：新しいコントローラとビューファイルを生成 `docker compose exec web rails model` ：Dockerコンテナ内のRailsアプリケーションで、Railsのgeneratorを使って新しいモデルを作成するコマンド。モデルはMVCフレームワークの中で、データの処理を担当する部分。 このコマンドを実行するとapp/modelsディレクトリに○○.rbというファイルが生成される。同時にdb/migrateディレクトリ配下にデータベースのテーブル構造を定義するマイグレーションファイルも生成される。 `docker compose exec web rails g migration` ：Dockerコンテナ内のRailsアプリケーション。Railsのgeneratorを使ってマイグレーションを生成するコマンド。`model`のコマンドでもマイグレーションファイルが作成されるが、モデルと対でない場合（カラムを追加したい・カラム名を変更したい等）で使用する。 マイグレーションファイルに必要な変更をした後、 `docker compose exec web rails db:migrate` を実行して変更をデータベースに適用する。 `docker compose exec web rails g controller` ：Railsのgeneratorを使ってコントローラーファイルを生成するコマンド。 config/application.rbの編集：不要なファイルを作らないようにする！ g.skip_routes true：ルーティングの記述を加えないようにする設定 g.helper false：ヘルパーファイルを自動生成しないようにする設定 g.test_framework nil：テストフレームワークを使わないようにする設定。 Lint：コードの品質やスタイルをチェックするツール。Rubyの場合はRubocop Rubocopを使用してエラーメッセージが出た場合、AI講師に以下のように聞いてみる ├── app #
